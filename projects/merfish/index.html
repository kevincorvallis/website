MERFISH · Mouse-Brain Atlas

Anatomy from RNA alone.

Read the RNA inside an intact mouse brain — one molecule at a time — then rebuild the brain from where those molecules sit.

MOUSE BRAIN · CORONAL / 483 genes / 83,546 cells / ~1.6×10⁸ transcripts / scanpy · squidpy · STalign

MERFISH (multiplexed error-robust fluorescence in situ hybridization) images individual RNA molecules right where they sit in tissue. Hundreds of genes per cell, positions kept — so the anatomy comes from the molecules themselves.

This project takes that raw scan all the way to a labeled, atlas-placed map — and proves every step on real data.

METHOD Reading a gene as a barcode

Every gene gets an error-correcting binary code.

MERFISH reads each gene as a binary codeword spread across many imaging rounds — the on/off pattern names the gene. The code self-corrects (16-bit, Hamming-distance-4), so one misread round can't change the answer. That's the "ER" in MERFISH.

Illustrative codewords (weight-4, Hamming distance 4). A real panel spreads hundreds of genes across the rounds, and misreading one round can't flip a call to another valid gene.

REGISTRATION The hardest part of a new scan

Putting a single 2D slice into a 3D brain.

Which brain region is each cell in? You have to fit a flat 2D slice into the 3D Allen atlas — right depth, right angle — before any label means anything. The pipeline does it (anchor → STalign warp → label lift-over) and adds what off-the-shelf tools skip: a per-cell confidence and a QC check.

STalign LDDMM, ground-truthed

On a slice cut from a known plane, STalign places it to 46.7 µm — under half a voxel — and gets 0.89 of per-cell labels right, in about two minutes on CPU. A real 2D→3D fit, not a stub.

Auto AP-anchor for STalign's init-sensitivity

STalign can misfire on its starting guess — a posterior slice landed 1.44 mm off. A training-free anchor finds the right depth first by sliding the slice against every atlas plane: ~10× better, and all three test slices align.

PROOF Tested on real data

It recovers published biology on real data.

The real test: take a published MERFISH dataset (73,626 cells) and run clustering blind. It recovers the authors' own cell types — ARI = 0.28 vs ~0 for random labels — with matching UMAP structure and real anatomy in the slice.

CONTEXT Where MERFISH sits in the field

One of three imaging-based platforms.

MERFISH is one of three imaging platforms that read single molecules in place. Imaging trades panel size for subcellular precision; sequencing trades precision for whole-transcriptome reach. No single winner.

SCOPE What's implemented vs what's wired-in

What's built, and what's wired in.

A light core runs in CI; the heavy atlases and engines plug in behind the same interface — no rewrite needed.

  • implementedScanpy pipeline, cell-type mapper, segmentation benchmark, and the registration core (geometry, label transfer, uncertainty, QC), all covered by offline pytest.
  • validatedAllen CCFv3 via brainglobe, and the STalign LDDMM deformable backend with a ground-truthed fit, in an isolated NumPy-2 / Python-3.12 environment.
  • wired-inDeepSlice, ANTs, and the Allen cell_type_mapper are stubbed behind the live interfaces with install hints. They need large downloads, not new architecture.

Honest caveat (also in the repo): the calibration and QC numbers use a synthetic registration error on real atlas geometry — so they test the math, not an engine's end-to-end accuracy. The STalign result is a real fit.

DECISIONS Why it's built this way

Engineering choices.

A confidence score on every call

Cell-type and region calls both ship with a calibrated confidence — trust the confident ones, flag the rest. Same trick, reused twice.

Light core, pluggable backends

Every stage has a no-download synthetic path, so the whole pipeline runs in CI. The real engines slot in behind the same signatures — the test suite is the spec.

Modern stack, on purpose

STalign pins an ancient numpy==1.23; a --no-deps install runs it on NumPy 2.4 / Python 3.12 with everything else. One environment, not two.
