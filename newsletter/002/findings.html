📖 Glossary — What do these terms mean?

Threshold — The sensitivity dial (0-1). Higher = stricter. At 0.6, the model only flags content it's 60%+ confident about.

Recall — What percentage of actual nudity scenes the model catches. 90% recall = catches 9 out of 10 real scenes.

Precision — Of everything the model flags, what percentage is actually nudity. 80% precision = 8 out of 10 flags are correct.

F1 Score — The balance between recall and precision. 1.0 = perfect. It penalizes models that sacrifice one for the other.

False Positive (FP) — When the model incorrectly flags non-nudity content as nudity. Think: a shirtless man at a pool party flagged as explicit.

False Negative (FN) — When the model misses actual nudity. A nude scene that passes through undetected.

AUC-PR — Area Under the Precision-Recall Curve. A single number (0-1) that summarizes overall model quality. Higher = better at distinguishing nudity from non-nudity.

Consensus — Requiring multiple models to agree before flagging. Like needing two referees to call a foul — reduces false calls.

Bimodality — Whether a model's scores cluster at two extremes (0 and 1) vs. being spread across the range. A bimodal model is more "binary" in its decisions.

Cohen's Kappa — Measures agreement between two models beyond what you'd expect by chance. 0 = random agreement, 1 = perfect agreement.

McNemar's Test — A statistical test that determines if two models perform significantly differently. p < 0.05 = the difference is real, not random.

Wasserstein Distance — Measures how different two score distributions are. Think of it as "how much work to transform one model's scoring pattern into the other's."

KDE — Kernel Density Estimation. A smooth curve showing where most scores concentrate. Like a smoothed histogram.

Methodology

A reproducible, multi-stage evaluation pipeline from raw frame-level model outputs to verified ground truth annotations with severity classifications.

End-to-End Pipeline Overview

1. Raw Data Sources

VionLabs API

Format: Frame-level nudity scores (0.0–1.0)
Rate: ~4.8 frames/sec (~35K frames per 2hr film)
Coverage: All 15 movies
XCD v1
Source: gs://xcd-dataset/inference_results/{video_id}_v1_full.json
Format: Frame-level nudity scores (0.0–1.0)
Rate: ~2.0 frames/sec (~16K frames per 2hr film)
Coverage: 7 movies (full comparison set)
XCD v2
Source: gs://xcd-dataset/inference_results/{video_id}_v2_full.json
Format: Frame-level nudity scores (0.0–1.0)
Rate: ~2.0 frames/sec (~16K frames per 2hr film)
Coverage: 7 movies (full comparison set)

2. Automated Scene Detection

Raw frame scores were processed through a peak detection algorithm to identify candidate nudity scenes:

  • Initial threshold: Frames with score > 0.30 flagged as candidates
  • Temporal clustering: Consecutive flagged frames within a 30-second gap grouped into a single detection cluster
  • Peak extraction: Within each cluster, the frame with the highest score becomes the peak frame (used for GIF extraction)
  • Multi-model merging: Clusters from VionLabs, XCD v1, and XCD v2 that overlap temporally are merged into a single scene candidate

This produced 327 candidate scenes across all 15 films.

3. Scene Consolidation

The 327 candidates were consolidated to avoid double-counting continuous nudity scenes that models fragmented:

  • Proximity merge: Scene candidates within 120 seconds of each other merged into a single ground truth scene
  • Rationale: Film nudity scenes often contain brief cuts away (dialogue, reaction shots) that cause models to report separate clusters for the same narrative scene
  • Result: 327 candidates → 214 verified ground truth scenes (35% consolidation rate)
Example: Wolf of Wall Street scene at 2:10
  VionLabs cluster: 2:10–2:35 (peak: 0.94)
  XCD v2 cluster: 2:28–3:13 (peak: 1.00)
  Gap: 0s (overlapping) → Merged into single scene 2:10–3:13

4. Manual Verification Process

Each of the 214 consolidated scenes was manually verified through a multi-step process:

  1. GIF extraction: 3-frame animated GIFs generated at each scene’s peak timestamp (start frame, peak frame, end frame)
  2. Visual inspection: Each GIF reviewed to confirm nudity is actually present and classify content type
  3. Film-knowledge verification: Cross-referenced against known film content from MPAA ratings, parent guides (Common Sense Media, IMDB Parents Guide), and film synopsis databases
  4. Severity assignment: Each scene assigned a severity code (S0–S4) based on visual content observed
  5. Quality flag: 66 scenes (31%) flagged for frame-level re-examination where model scores conflicted or GIF was ambiguous

5. Severity Classification System

A 5-level severity scale was designed to classify scene content:

Code Level Description Example
S0 Safe / False Positive No nudity present; model triggered on skin tones, swimwear, or non-nude content Beach volleyball scene in Top Gun
S1 Skin Exposure Significant skin visible but no nudity; shirtless, underwear, suggestive framing Locker room scene with towels
S2 Non-Sexual Nudity Nudity in non-sexual context; artistic, medical, changing, bathing Character getting dressed after bath
S3 Sexual Nudity Nudity in a sexual context; love scenes, seduction, intimate moments Love scene in Basic Instinct
S4 Explicit / Graphic Graphic nudity; full frontal, extended explicit scenes, pornographic framing Extended scenes in Wolf of Wall Street

6. Cross-Validation & Quality Assurance

Ground truth annotations were cross-validated against multiple external sources:

  • MPAA Ratings: Scene counts cross-checked against rating justifications (e.g., “Rated R for strong sexual content, nudity”)
  • IMDB Parents Guide: “Sex & Nudity” sections used to verify scene descriptions and approximate timestamps
  • VionLabs Frame Data: High-confidence VionLabs detections (>0.9) used as initial anchor points for scene boundaries
  • PG/PG-13 Baseline: 3 non-R-rated films (Top Gun PG, Top Gun: Maverick PG-13, A Quiet Place PG-13) included as negative controls — expected to have mostly S0/S1 scenes

v1 Is Statistically Proven Inferior

Both McNemar's tests involving v1 show p < 0.001. The model misses 65 of 214 verified scenes (48.5% miss rate) even at its optimal threshold of 0.3.

Cohen's kappa between v2 and v1 is only 0.087 -- barely above chance agreement. These models are looking at fundamentally different things.

VionLabs and v2 Are Complementary

Kappa of 0.280 (fair agreement) means they agree on the easy cases but have distinct detection capabilities. This complementarity is exactly what makes AND-consensus so powerful -- it eliminates false positives from both models.

💡 What This Means

McNemar's test is a statistical method that tells us whether two models perform differently enough that the difference isn't just luck. The p-value tells us the probability that the difference is random — below 0.05 means it's real. VionLabs and v2 are statistically equivalent (p=0.92), meaning they catch the same scenes. But both are significantly better than v1 (p<0.001) — v1's poor performance is not a fluke.
